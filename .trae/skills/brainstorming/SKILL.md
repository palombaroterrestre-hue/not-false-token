# Brainstorming Workflow

Use this skill to ensure alignment on design and architecture before implementation.

## 9-Step Process

1.  **Understand Goals**: Clarify what the user wants to achieve.
2.  **Gather Context**: Identify constraints, libraries, and existing code.
3.  **Ideation**: Propose multiple high-level approaches.
4.  **Refinement**: Narrow down to the best approach.
5.  **Technical Design**: Detail the architecture, file structure, and logic.
6.  **Edge Cases**: Identify potential issues and solutions.
7.  **Review**: Present the final design to the user.
8.  **Approval (HARD GATE)**: Wait for user confirmation.
9.  **Implementation**: Proceed with coding only after approval.

<HARD-GATE>
Do NOT invoke any implementation skill, write any code, scaffold any project, or take any implementation action until you have presented a design and the user has approved it.
</HARD-GATE>
